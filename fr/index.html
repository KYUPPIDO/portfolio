Groupie Tracker

go · api rest · openstreetmap · équipe de trois

Une application web de découverte de concerts, construite avec deux camarades. Au démarrage, le serveur Go récupère quatre sources d'API éparpillées, les artistes, les salles, les dates et les relations entre elles, les fusionne côté serveur, et sert à la fois les pages et sa propre petite API JSON.

Ce que je défendrais en revue de code : un panneau de filtres où quatre critères se combinent (nombre de membres, année de création, premier album, lieu) sans que la page ressemble jamais à une requête de base de données, et une carte des concerts qui géocode les salles via OpenStreetMap, avec un cache pour ne jamais chercher deux fois la même adresse.

Net Cat

go · sockets tcp · goroutines · concurrence

Une reconstruction de nc en salon de discussion : un serveur Go, jusqu'à dix clients, du TCP brut. Chaque client vit dans sa propre goroutine, un mutex protège ce qu'ils partagent. Les nouveaux venus choisissent un nom unique, reçoivent l'historique, puis chaque message atteint chaque client connecté, horodaté et consigné dans un journal de session.

Le projet a aussi gagné ses petits conforts : des commandes /rename et /list, un admin qui peut rendre muet ou bannir, et assez de soin sur les codes d'échappement du terminal pour qu'un message entrant n'abîme jamais la ligne en cours de frappe. C'est le projet qui m'a appris que la concurrence est moins une affaire de vitesse que de savoir qui parle quand.
